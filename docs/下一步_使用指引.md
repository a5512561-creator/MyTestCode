# 下一步：使用指引（Outlook 已可建立事件）

目前你已完成：Planner 匯出任務 → nextweek 分析 → schedule-nextweek 寫入 JSON → 手動執行 Flow 在 Outlook 建立事件。以下為建議的下一步。

---

## 1. 正式使用排程（取消筆數限制）

測試沒問題後，改為排入**全部**下週待排任務：

- 在 **.env** 將 `SCHEDULE_LIMIT=3` **刪除**或改為 **`SCHEDULE_LIMIT=0`**（0 = 不限制）。
- 之後執行 `schedule-nextweek` 會把「延續到下週」與「下週到期需準備」的任務全部寫入 `schedule_requests.json`，手動執行 Flow 後會建立對應數量的 Outlook 事件。

---

## 2. 每週建議流程（固定節奏）

可依下面順序做成每週習慣：

| 時機 | 動作 |
|------|------|
| **週末或週一前** | 1. Power Automate 手動執行 **GotPlannerTasks**（更新 tasks_output.json）<br>2. 本機執行 `py -X utf8 src/agent.py nextweek` 看本週／下週任務與 Triage<br>3. 在 Planner 補齊缺估時（標題加 [2h] 等）、必要時調整到期日<br>4. 再執行一次 GotPlannerTasks（可選，若剛改過任務）<br>5. 執行 `py -X utf8 src/agent.py schedule-nextweek` 產生 schedule_requests.json<br>6. 等 OneDrive 同步後，手動執行 **從檔案建立 Outlook 事件** Flow<br>7. 到 Outlook 確認事件 |
| **週三下班前** | 整理 weekly status（目前可手動寫 OneNote；若之後有建 OneNote Flow 可再串接） |

---

## 3. 可選進階：避開會議與工作時段

目前排程是從「下週一 9:00」依序往後排，**不會**避開既有會議、午休、晚餐或六日。

若要做「只排進空檔、遵守工作時段、六日不排、排不進的任務提醒」，需要：

- 實作計畫 **「行事曆排程避開會議與工作時段」**：
  - 新增 **Flow：取得下週行事曆事件**（手動觸發 + 寫入檔案），供 Agent 讀取既有會議。
  - 在 Agent 端依工作時段參數（上班時間、午休、晚餐、工作日）算出空檔，只對空檔寫入要建立的事件；排不進的任務列為「未排入」並提醒。
- 相關參數與流程見計畫檔（工作時段、一週起訖、CALENDAR_BUSY_FILE 等）。

若你希望先維持目前「依序排入」即可，可暫不實作，之後有需要再做。

---

## 4. 可選：OneNote 週報（Flow 3）

你提到每週三下班前整理 weekly status；若希望由 Agent／Flow 協助產出週報頁面：

- 需在 Power Automate 建立 **Flow 3：建立 OneNote 週報頁面**（手動或 HTTP 觸發，依公司授權決定）。
- 在 .env 設定 **FLOW_ONENOTE_URL** 或對應的檔案／觸發方式。
- 執行 `py -X utf8 src/agent.py onenote` 會組好週報內容並呼叫該 Flow（或寫入檔案供 Flow 讀取）。

目前專案已有週報相關程式與文件骨架，可依實際 OneNote 與授權再補完。

---
