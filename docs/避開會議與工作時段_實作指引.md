# 避開會議與工作時段：實作指引

分兩部分：**Part A** 在 GotPlannerTasks Flow 裡一併抓取行事曆（上週+下週）並寫入同一檔案；**Part B** 在 Agent 端讀取行事曆、計算工作時段與空檔，只把任務排進空檔並回報未排入。

---

## Part A：在 Flow 1 加入行事曆並寫入 tasks_output.json

目標：讓 **tasks_output.json** 除了 `tasks`、`buckets` 外，多一個 **`calendarEvents`**（上週+下週行事曆事件），Agent 用來當「既有會議／忙碌時段」與週報會議列表。

**具體步驟（取得行事曆、Compose 加入 calendarEvents、寫入檔案）請直接參閱 [Flow1.md](Flow1.md) 的「步驟 4：取得行事曆（上週 + 下週）」、「步驟 5：Compose」、「步驟 6：寫入 OneDrive 檔案」與「十一、檢查輸出格式」。**

---

## Part B：Agent 端（讀取 calendarEvents、工作時段、空檔、未排入提醒）

Part A 完成後，同一份 **tasks_output.json** 已包含下週行事曆。接下來在 Agent 端要做：

1. **讀取 calendarEvents**：從 `TASKS_INPUT_FILE` 的 JSON 讀取 `calendarEvents`（若無或空則視為無會議）。
2. **工作時段參數**：從 .env 讀取 `WORK_DAY_START`、`WORK_DAY_END`、`LUNCH_START`／`LUNCH_END`、`DINNER_START`／`DINNER_END`、`WORK_DAYS`（一週起訖若已實作則用 `WEEK_START_DAY`／`WEEK_END_DAY` 決定「下週」區間）。
3. **可工作時段**：依參數產出「下週」內每個工作日的可排時段（扣除午休、晚餐），六日不排。
4. **busy → free**：可工作時段減去 `calendarEvents` 的 start～end，得到空檔列表。
5. **排程**：依序把任務放進空檔（每筆用估時）；放不進的任務列入 **not_scheduled** 並在輸出中提醒。
6. **寫檔**：只把「有排進空檔」的任務寫入 **schedule_requests.json**（或維持現有寫入邏輯，但在程式中區分 created / not_scheduled）。

Part B 的具體實作（含 `build_work_windows`、`busy_to_free`、與 `write_schedule_requests_to_file` 的整合）會依你目前的 `powerautomate_client` 結構接著改程式並補上 .env 範例。

---

## 建議執行順序

1. **先完成 Part A**：依 [Flow1.md](Flow1.md) 在 GotPlannerTasks 加上取得行事曆、Compose 加入 `calendarEvents`、確認 tasks_output.json 含三欄位。
2. **再進行 Part B**：在專案裡實作工作時段、空檔計算、以及只排進空檔並回報未排入的邏輯。
