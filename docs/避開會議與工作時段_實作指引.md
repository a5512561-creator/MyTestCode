# 避開會議與工作時段：實作指引

分兩部分：**Part A** 在 GotPlannerTasks Flow 裡一併抓取下週行事曆並寫入同一檔案；**Part B** 在 Agent 端讀取行事曆、計算工作時段與空檔，只把任務排進空檔並回報未排入。

---

## Part A：修改 GotPlannerTasks Flow（同一個 Flow 抓 Planner + 下週行事曆）

目標：讓 **tasks_output.json** 除了 `tasks`、`buckets` 外，多一個 **`calendarEvents`**（下週行事曆事件），Agent 用來當「既有會議／忙碌時段」。

### A.1 在「List buckets for a plan」之後新增：取得行事曆事件（上週 + 下週）

1. 開啟你的 **GotPlannerTasks** Flow。
2. 在 **List buckets for a plan** 與 **Compose** 之間，新增步驟。
3. 搜尋 **Office 365 Outlook**（或你用的 Outlook 連接器）→ 選 **取得行事曆檢視中的事件**（Get calendar view of events）或 **List events in a calendar view**。

**參數：** 一次取得**上週與下週**（供排程避開會議 + 週報會議列表／統計）。在 **Start time**、**End time** 欄位點 **fx** 貼上以下運算式：

| 參數 | 值（貼到 fx 後按 OK） |
|------|------------------------|
| **Calendar** | 選 **Calendar**（主要行事曆）。 |
| **Start time** | 上週一 00:00 UTC：<br>`concat(formatDateTime(addDays(startOfWeek(utcNow(), 'Monday'), -7), 'yyyy-MM-dd'), 'T00:00:00Z')`<br>若報錯可試 [Flow1_行事曆_上週與下週日期運算式.md](Flow1_行事曆_上週與下週日期運算式.md) 的寫法 B 或 C。 |
| **End time** | 下週日 23:59:59 UTC：<br>`concat(formatDateTime(addDays(startOfWeek(utcNow(), 'Sunday'), 7), 'yyyy-MM-dd'), 'T23:59:59Z')` |

更多寫法（僅日期、或欄位型別不同）見 **[Flow1_行事曆_上週與下週日期運算式.md](Flow1_行事曆_上週與下週日期運算式.md)**。

4. 此步驟輸出會有一串事件（陣列），每筆通常有 **start**、**end**（或 **Start**、**End**）等。記下此步驟名稱，例如 **Get_calendar_view_of_events**。Agent 會依 start/end 判斷事件屬於上週或下週。

### A.2 修改 Compose：加入 calendarEvents

原本 Compose 組的是 `{ "tasks": ..., "buckets": ... }`，要改成 **三個** 鍵：`tasks`、`buckets`、**calendarEvents**。

**做法 A（鍵值 + 動態內容）：**

1. 點開 **Compose** 的 **Inputs**。
2. 新增第三組參數：
   - **鍵**：`calendarEvents`
   - **值**：動態內容 → 選 **Get calendar view of events**（或你剛加步驟的名稱）→ 選 **value**（事件陣列）。若沒有 **value**，改選 **body** 或畫面上代表「事件清單」的欄位。
3. 儲存。

**做法 B（運算式）：**

若用運算式組一整份 JSON，在原有 `concat` 裡多加 `,"calendarEvents":', string(body('Get_calendar_view_of_events')?['value']), '}'` 之類（依實際步驟名稱與輸出結構改）。例如：

```text
json(concat('{"tasks":', string(body('List_my_tasks')?['value']), ',"buckets":', string(body('List_buckets_for_a_plan')?['value']), ',"calendarEvents":', string(body('Get_calendar_view_of_events')?['value']), '}'))
```

（將 `Get_calendar_view_of_events` 換成你畫面上的步驟名稱；若事件在 **body** 或別欄，改 `['value']` 為對應欄位。）

### A.3 寫入檔案（不變）

**Create file** 仍寫入同一個 **tasks_output.json**，**File Content** 改為選**更新後的 Compose** 的 **Output**（已含 tasks、buckets、calendarEvents）。

### A.4 檢查輸出格式

手動執行 Flow 一次，到本機打開同步後的 **tasks_output.json**，確認有：

- `tasks`：陣列
- `buckets`：陣列  
- `calendarEvents`：陣列，每筆事件至少要有 **start**／**end**（或 **Start**／**End**，Agent 可再對應）

若 Outlook 回傳的欄位是 **Start**、**End**（大寫），Agent 端會依此讀取；若為 **start**、**end** 也可。若有 **start.dateTime**、**end.dateTime**，Agent 需用該路徑讀取（實作時會一併處理）。

## Part B：Agent 端（讀取 calendarEvents、工作時段、空檔、未排入提醒）

Part A 完成後，同一份 **tasks_output.json** 已包含下週行事曆。接下來在 Agent 端要做：

1. **讀取 calendarEvents**：從 `TASKS_INPUT_FILE` 的 JSON 讀取 `calendarEvents`（若無或空則視為無會議）。
2. **工作時段參數**：從 .env 讀取 `WORK_DAY_START`、`WORK_DAY_END`、`LUNCH_START`／`LUNCH_END`、`DINNER_START`／`DINNER_END`、`WORK_DAYS`（一週起訖若已實作則用 `WEEK_START_DAY`／`WEEK_END_DAY` 決定「下週」區間）。
3. **可工作時段**：依參數產出「下週」內每個工作日的可排時段（扣除午休、晚餐），六日不排。
4. **busy → free**：可工作時段減去 `calendarEvents` 的 start～end，得到空檔列表。
5. **排程**：依序把任務放進空檔（每筆用估時）；放不進的任務列入 **not_scheduled** 並在輸出中提醒。
6. **寫檔**：只把「有排進空檔」的任務寫入 **schedule_requests.json**（或維持現有寫入邏輯，但在程式中區分 created / not_scheduled）。

Part B 的具體實作（含 `build_work_windows`、`busy_to_free`、與 `write_schedule_requests_to_file` 的整合）會依你目前的 `powerautomate_client` 結構接著改程式並補上 .env 範例。

---

## 建議執行順序

1. **先完成 Part A**：在 GotPlannerTasks 加上「取得下週行事曆」步驟、Compose 加入 `calendarEvents`、確認 tasks_output.json 長相。
2. **再進行 Part B**：在專案裡實作工作時段、空檔計算、以及只排進空檔並回報未排入的邏輯。

你完成 Part A 後跟我說一聲（或貼一段 tasks_output.json 的範例，遮掉敏感內容即可），我再依實際的 **calendarEvents** 欄位名稱與格式，幫你寫 Part B 的讀取與計算邏輯。
