# 當組織要求管理員核准時（realtek.com）

若登入時出現「需要管理員核准；只有系統管理員可以授與此應用程式的權限」，表示貴公司已設定：**所有應用程式（含用你的帳號登入）都需管理員同意**，才能存取公司資料。

---

## 方案 A：請管理員核准一次（建議優先）

請 IT／管理員對你的應用程式 **PaddyPlannerAgent** 做 **一次**「授與管理員同意」：

1. 他們用具備管理員權限的帳號登入 **Azure 入口網站**。
2. **Azure Active Directory** → **企業應用程式** → 搜尋 **PaddyPlannerAgent**。
3. 左側 **權限** → 按 **「授與 [組織名稱] 管理員同意」**。
4. 核准後，你再用 **裝置碼登入**（`.env` 設 `DEVICE_CODE_FLOW=true`、註解 `CLIENT_SECRET`）即可正常使用 Agent。

只需核准一次，之後不必再申請。

---

## 方案 B：不用你的 App，改由 Power Automate 當後端（不需管理員核准）

若暫時無法取得管理員同意，可以 **不讓 Python 直接呼叫 Graph**，改成由 **Power Automate** 代勞（Power Automate 的連接器通常已被組織核准）。

### 概念

- 在 Power Automate 建立幾個 **以 HTTP 觸發** 的 Flow，分別負責：
  - 列出 Planner 任務
  - 建立 Outlook 行事曆事件
  - 建立 OneNote 週報頁面
- Python Agent 只呼叫這些 Flow 的 **HTTP 觸發網址**（必要時帶參數或 body），不再使用 Azure 應用程式註冊與 Graph token。

### 優缺點

- **優點**：不需管理員核准你的 App；Power Automate 你已有權限即可。
- **缺點**：需在 Power Automate 設計與維護這些 Flow，並把觸發網址保管在 `.env`。

### 實作大綱

1. **Flow 1：取得 Planner 任務**
   - 觸發：**當收到 HTTP 要求時**（方法 GET 或 POST）。
   - 動作：Planner → **列出計畫的任務** 或 **取得我的任務**（依你現有權限選），計畫 ID 可寫死在 Flow 或從 HTTP 查詢參數讀取。
   - 回應：把任務清單回傳給呼叫端（JSON）。

2. **Flow 2：建立行事曆事件**（可選）
   - 觸發：**當收到 HTTP 要求時**。
   - 從 body 讀取：標題、開始時間、結束時間等。
   - 動作：Outlook 行事曆 → **建立事件**。
   - 回應：成功／失敗或事件 ID。

3. **Flow 3：建立 OneNote 週報**（可選）
   - 觸發：**當收到 HTTP 要求時**。
   - 從 body 讀取：標題、內容（HTML 或純文字）。
   - 動作：OneNote → **建立頁面**。
   - 回應：成功／失敗或頁面 URL。

4. **在 Python 專案**
   - 在 `.env` 新增例如：`PLANNER_FLOW_URL=...`、`CALENDAR_FLOW_URL=...`、`ONENOTE_FLOW_URL=...`。
   - 修改 `src/planner_client.py`（或新建 `src/powerautomate_client.py`）：改為用 `requests.get(PLANNER_FLOW_URL)` 取得任務，不再呼叫 Graph。
   - 同樣方式改行事曆、OneNote 的呼叫，改打對應的 Flow URL。

這樣 Agent 的「大流程」不變（整理任務 → 排程 → 週報），只是 **資料來源與寫入改由 Power Automate 執行**，Python 只負責邏輯與呼叫 Flow。

---

## 小結

| 做法 | 是否需要管理員 | 說明 |
|------|----------------|------|
| **方案 A** | 需要一次核准 | 請管理員對 PaddyPlannerAgent 授與同意，之後用裝置碼登入即可。 |
| **方案 B** | 不需要 | 用 Power Automate HTTP Flow 當後端，Python 只呼叫 Flow，不直接呼叫 Graph。 |

若你決定採用方案 B，我可以依你現有的 `planner_client` / `calendar_client` / `onenote_client` 介面，幫你規劃要建哪些 Flow、以及 Python 該怎麼改寫成呼叫這些 URL。
